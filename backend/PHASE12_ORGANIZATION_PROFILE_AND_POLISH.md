# Phase 12 - Organization Profile, Report Branding, and UI/UX Polish

## Organization Profile

Phase 12 adds a reusable Organization Profile module so each CCMS installation can configure its own report identity without hardcoded organization-specific names.

Fields:

- organization_name
- short_name
- logo_path stored internally only
- address
- city
- district
- province
- country
- phone
- email
- website
- registration_no
- ntn_or_tax_no
- report_footer_text
- report_watermark_text
- primary_color
- secondary_color
- authorized_signatory_name
- authorized_signatory_designation
- is_active

If no profile exists, the system safely falls back to:

- Organization Name: Child Care Management System
- Short Name: CCMS
- Address: Not configured
- Logo: no logo / short-name fallback

## Backend Endpoints

- `GET /organization-profile`
- `PUT /organization-profile`
- `POST /organization-profile/logo`
- `GET /organization-profile/logo`
- `DELETE /organization-profile/logo`

Only Admin can update the profile or upload/delete the logo. Operational roles can read public branding for report display. Absolute local filesystem paths are never returned to the frontend.

## Logo Upload

Supported logo formats:

- PNG
- JPG
- JPEG
- WEBP

Logo upload uses the existing safe upload utilities and backend upload size limit.

## Report Branding Behavior

PDF reports now resolve Organization Profile branding first and fall back to the CCMS default branding if no profile is configured.

Every branded PDF includes:

- Logo or short-name fallback
- Organization name
- Address/contact line
- Report title
- Generated by
- Generated at
- Filters or period where applicable
- Footer text
- System-generated notice
- Page number

Updated PDF paths include children, full child profile, sponsors, sponsorships, accommodation, medical, education, case management, daily attendance, monthly attendance, visitors, child visits, daily visitor register, monthly visitor register, users, and audit summary exports.

## PDF Design Rules

- Professional header with organization identity
- Summary cards for list reports where useful
- Clean table styling and spacing
- Friendly labels instead of raw technical field names
- Empty values shown as `-`
- Sensitive fields filtered from general reports

Sensitive fields excluded by the PDF service include password, token, hash, CNIC, passport, mobile, address, diagnosis, treatment, confidential, and restricted fields.

## Manual Workflow Checklist

1. Start backend.
2. Login as Admin.
3. Open Organization Profile.
4. Save organization name, address, contact, colors, and footer text.
5. Upload logo.
6. Download Children PDF.
7. Download Daily Attendance PDF.
8. Download Visitor Register PDF.
9. Confirm organization branding appears.
10. Confirm sensitive fields are not printed.
11. Login as Viewer and confirm profile editing is not available.
12. Confirm existing modules still work.

## Known Limitations

- Direct public unauthenticated logo serving is intentionally not enabled.
- Watermark text is stored for branding configuration but not rendered as a page watermark in this safe polish pass.
